## 二、LLM 大模型概述

### 2.1 基本概念

大型语言模型（Large Language Model），基于海量数据进行学习的模型，核心机制是"下一个字词预测"。

### 2.2 技术选型

| 类型 | 优点 | 缺点 |
|------|------|------|
| **在线大模型**（GPT、Gemini、GLM、文心一言） | 性能优越、成本低（0.005-0.008元/千token）、快速上手 | 数据安全风险、受制于人 |
| **开源大模型**（Llama、Qwen、Baichuan） | 完全自主可控、定制性强、数据安全 | 需要大量算力和开发成本 |

### 2.3 主流开源模型系列

- **Llama系列**：Meta开源，当前最好开源大模型之一
- **Qwen系列**：阿里通义千问，国内开源大模型领袖

### 2.4 Prompt 工程

提示工程是优化提示词以获得更好结果的工程技术。

**核心原则**：
- 清晰易懂
- 提供例子和锁定上下文
- 明确步骤
- 准确表达意图

**消息类型**：
- `system`：系统指令，优先级最高
- `user`：用户输入
- `assistant`：模型回复

## 三、Ollama 本地部署

### 3.1 简介

Ollama 是一个开源的大型语言模型服务工具，专为在服务器上便捷部署和运行LLMs而设计。

**特点**：
- 开源免费
- 简化部署
- 轻量级与可扩展
- API支持
- 预构建模型库

### 3.2 常用命令

| 命令 | 描述 |
|------|------|
| `ollama list` | 列出本地模型 |
| `ollama pull` | 拉取模型 |
| `ollama run` | 运行模型 |
| `ollama ps` | 查看当前加载的模型 |
| `ollama serve` | 启动HTTP服务 |
| `ollama rm` | 删除模型 |
| `ollama create` | 从Modelfile创建自定义模型 |

### 3.3 自定义模型（Modelfile）

Modelfile 是 Ollama 创建模型的配置文件，关键指令：

- `FROM`：基础模型
- `SYSTEM`：系统消息
- `PARAMETER`：运行参数（temperature、top_p、top_k等）
- `TEMPLATE`：提示模板
- `MESSAGE`：消息历史

### 3.4 模型下载

使用 ModelScope 下载模型：

```python
from modelscope import snapshot_download

name = 'qwen/Qwen2.5-7B-Instruct-GGUF'
model_path = r'C:\huggingface'
model_dir = snapshot_download(
    name,
    cache_dir=model_path,
    revision='master',
    allow_file_pattern="Qwen2.5-7B-Instruct.Q5_0*.gguf"
)
```

## 四、大模型开发

### 4.1 大模型分类

**按数据类型**：
- 语言大模型（NLP）：GPT系列、文心一言
- 视觉大模型（CV）：VIT系列、华为盘古
- 多模态模型：DALL-E、悟空画画

**按应用层级**：
- L0：通用大模型
- L1：行业大模型
- L2：垂直大模型

### 4.2 Embedding 向量化

**概念**：将文本转换为向量数值，捕获语义信息。

**应用**：相似文本检索、聚类分析、推荐系统等。

> 万物皆可Embedding

### 4.3 向量数据库

专门用于存储和处理向量数据的数据库，能够快速进行相似度搜索。

**常见选择**：
- Milvus
- Chroma
- Weaviate
- Faiss
- PGVector
- Elasticsearch

## 五、LangChain 框架

### 5.1 简介

LangChain 是一个强大的框架，帮助开发者使用语言模型构建端到端应用程序。

**核心理念**：为各种LLMs提供通用接口，降低开发成本。

### 5.2 核心组件

| 组件 | 功能 |
|------|------|
| **Model I/O** | 管理语言模型、输入提示、输出解析 |
| **Data Connection** | 管理向量数据存储、内容加载、向量检索 |
| **Memory** | 保持应用状态，存储对话历史 |
| **Chains** | 构建调用序列，串联各组件 |
| **Agents** | 串联工具，让LLM自主决策 |
| **Callbacks** | 记录中间步骤，便于日志追踪 |

### 5.3 核心模块

#### Model（模型）
- **LLM**：文本字符串输入，文本字符串输出
- **Chat Model**：聊天消息列表输入/输出

#### Prompt（提示词）
- **PromptTemplates**：可复用的提示模板
- **ChatPromptTemplates**：聊天模型提示模板
- **Example Selectors**：示例选择器

#### Memory（记忆）
- 保持对话状态
- 支持短期/长期记忆
- 可返回字符串或消息列表

#### Agent（代理）
- 使用LLM作为思考工具
- 自主选择工具完成任务
- 动作代理 vs 计划-执行代理

### 6.2 自定义 LLM 类

```python
from transformers import AutoModelForCausalLM, AutoTokenizer
from langchain.llms.base import LLM

class Qwen(LLM):
    def __init__(self, model_name="Qwen/Qwen2.5-7B-Instruct"):
        self.model = AutoModelForCausalLM.from_pretrained(
            model_name, torch_dtype="auto", device_map="auto"
        )
        self.tokenizer = AutoTokenizer.from_pretrained(model_name)
    
    def _call(self, prompt, stop=None, run_manager=None):
        messages = [
            {"role": "system", "content": "You are Qwen, a helpful assistant."},
            {"role": "user", "content": prompt}
        ]
        text = self.tokenizer.apply_chat_template(messages, tokenize=False, add_generation_prompt=True)
        model_inputs = self.tokenizer([text], return_tensors="pt").to(self.model.device)
        generated_ids = self.model.generate(**model_inputs, max_new_tokens=512)
        response = self.tokenizer.batch_decode(
            generated_ids[:, len(model_inputs.input_ids[0]):], 
            skip_special_tokens=True
        )[0]
        return response
    
    @property
    def _llm_type(self):
        return "Qwen"
```
